XPU 芯片光谱看板

四个视角比较主流 AI 处理器：定性光谱定物种 → 量化坐标定档位 → Scale-up 域定真实战力 → TCO 代理定性价比 · 数据更新见页脚

图A · 定性光谱：它是什么物种

横轴 = 灵活性↔专用度（研究员定性判断，C级）；纵轴 = 单芯片功耗量级（对数示意），自然分出云侧/端侧两个战场；箭头 = 同族最近数代迭代方向。

图B · 量化坐标：进化到第几档

横轴 = 单芯片算力（FP8当量 TFLOPS，对数）；纵轴 = 内存带宽（TB/s，对数）；气泡大小 = 显存容量。斜向阴影带 = 算力:带宽均衡走廊（每 TFLOP 配约 1–7 GB/s）。空心点 = 非FP8口径折算。

模块C · Scale-up 域对比：真实的比较单位

大模型训练/推理受集合通信约束，单芯片纸面算力必须放进「一个互联域能圈住多少芯片、每芯片多大互联带宽」的框架里才有意义。域的组织方式（铜/光/OCS）差异极大，这正是单芯片对比失真的主因。
口径警告：各家「域」定义不同（NVIDIA=单机柜72卡全互联；谷歌=OCS连接的9216卡Pod；华为=16机柜全光384卡）。域算力可比，但域间跨域扩展（scale-out）能力此表不覆盖。

模块D · TCO 代理：性价比的可测算部分

完整 TCO =（芯片+系统资本开支）÷（寿命×利用率）+ 电费×PUE + 网络/运维。自研芯片的内部成本、良率、真实利用率均不可得，因此用「云租赁市场价」作市场化 TCO 代理，归一为 $/PFLOP·hr（FP8当量）。全部 C 级、随行就市，仅看相对档位。
三个陷阱：① 挂牌价≠成交价（TPU 大客户承诺价远低于挂牌）；② $/PFLOP 未计入软件生态带来的实际利用率差（CUDA 生态下 GPU 有效利用率通常更高）；③ 峰值算力≠有效算力（MFU 各家差异 30–60%）。此表回答「谁在降价抢份额」，不直接回答「谁更划算」。
维护说明（新卡迭代后如何刷新）

本页数据-逻辑分离：全部数据在页面源码顶部 ██ 数据区 ██ 内的四个常量（META / CHIPS / RACKS / TCO），渲染逻辑在页面底部，刷新只改数据区。步骤：

1. 新芯片发布 → 在 CHIPS 追加一行：填 fam（同族代际链自动连箭头）、spec_x（定性光谱位，参考同族上代 ±3）、power_w（图A）、tflops/bw/hbm（图B，FP8当量口径；无FP8口径的填折算值并设 hollow:true）、grade（A/B/C）。lx/ly 是图B标签偏移，重叠时微调。

2. 机柜/域级新品 → 改 RACKS；租赁价季度刷一次 → 改 TCO（记得同步改 note 里的口径说明）。

3. 改完更新 META.updated，浏览器打开自检两图无标签重叠，然后跑研究站 run_daily.ps1（或手动刷新研究站.bat）同步公网副本。

4. 数据纪律：图B只收有公开 spec 的芯片（B级优先）；估算值必须在 prec/名字里带「估」；定性 spec_x 永远是 C 级，不写来源。
