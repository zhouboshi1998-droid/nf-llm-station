XPU 芯片光谱看板

四个视角比较主流 AI 处理器：定性光谱定物种 → 量化坐标定档位 → Scale-up 域定真实战力 → TCO 代理定性价比。附录含主要商品芯片信息卡。

研究结论
先给判断，图表在后作证据。标「观点」的是研究判断（含 AI 观点，置信中，依据见对应图表），标「事实」的可溯源。
图A · 定性光谱：它是什么物种
横轴 = 灵活性↔专用度（研究员定性判断，C级）；纵轴 = 单芯片功耗量级（对数示意），自然分出云侧/端侧两个战场；箭头 = 同族最近数代迭代方向。
图B · 量化坐标：进化到第几档
横轴 = 单芯片算力（FP8当量 TFLOPS，对数）；纵轴 = 内存带宽（TB/s，对数）；气泡大小 = 显存容量；斜向阴影带 = 算力:带宽均衡走廊（每 TFLOP 配约 1–7 GB/s）；空心点 = 非FP8口径折算；实色 = 最新一代，淡色 = 历史型号；点任意点看芯片详情。
Scale-up 域对比：真实的比较单位
大模型负载受集合通信约束，采购和比较的真实单位是「一个互联域能圈住多少芯片、每芯片多大互联带宽」。域的组织方式（铜/光/OCS）差异极大，这正是单芯片对比失真的主因。
口径警告：各家「域」定义不同（NVIDIA=单机柜72卡全互联；谷歌=OCS连接的9216卡Pod；华为=16机柜全光384卡）。域算力可比，跨域 scale-out 能力此表不覆盖。
TCO 代理：性价比的可测算部分
完整 TCO =（芯片+系统资本开支）÷（寿命×利用率）+ 电费×PUE + 网络/运维。自研芯片内部成本、良率、真实利用率不可得，故用云租赁市场价作代理，归一为 $/PFLOP·hr（FP8当量）。全部 C 级、随行就市，只看相对档位。
三个陷阱：① 挂牌价≠成交价（TPU 大客户承诺价远低于挂牌）；② 未计入生态带来的实际利用率差（CUDA 生态下 GPU 有效利用率通常更高）；③ 峰值算力≠有效算力（MFU 差异 30–60%）。此表回答「谁在降价抢份额」，不直接回答「谁更划算」。
附录 · 主要商品芯片信息卡
「商品芯片」= 对外销售/出租的加速器（云厂纯自用的 TPU/Trainium/Maia/MTIA 不在此列，见图A/图B）。圆点颜色沿用全页阵营图例；标「估」处为 C 级。
维护说明（新卡迭代后如何刷新）

本页数据-逻辑分离：全部数据在源码顶部 ██ 数据区 ██ 的五个常量（META / CHIPS / RACKS / TCO / CONCLUSIONS+COMMODITY），渲染逻辑在页面底部，刷新只改数据区：

1. 新芯片发布 → CHIPS 追加一行：填 fam（同族自动连箭头）、spec_x（定性光谱位，参考同族上代 ±3）、power_w（图A）、tflops/bw/hbm（图B，FP8当量；非FP8口径填折算值并设 hollow:true）、grade。标签重叠用 lx/ly（图B）与 laxA/layA/aend/nolabel（图A）微调。

2. 商品芯片同步补一张 COMMODITY 信息卡；机柜/域级新品改 RACKS；租赁价季度刷 TCO；结论有更新改 CONCLUSIONS（判断层，改动需研究员过目）。

3. 改完更新 META.updated，浏览器自检无标签重叠，跑研究站 run_daily.ps1（或手动刷新研究站.bat）同步公网副本 xpuboard.html。

4. 数据纪律：图B只收有公开 spec 的芯片（B级优先）；估算值必须带「估」；定性 spec_x 永远 C 级。
